## TASK-007.P2 Implementation notes

- Create a temporary external-style fixture outside the Atom checkout, preferably under a short path such as `/tmp/atom-ext-fixture-<suffix>` to avoid Unix socket path limits.
- The fixture should have its own `deps.edn`, `src`, and `test` directories.
- Its `deps.edn` should add this Atom checkout as a test/dev `:local/root` dependency, not rely on a globally installed Atom or the user's default config-dir.
- Include tests that prove:
  - pure library code tests run without daemon assumptions
  - the author test JVM can require Atom namespaces, including `atom.test.alpha`
  - `atom.test.alpha` can start a disposable daemon world for the fixture
  - the fixture library is approved via generated `libs.edn`
  - daemon-side `(atom.libs.alpha/sync!)` and `(atom.libs.alpha/use! ...)` can load/activate the fixture library
  - both `:storage :sqlite-file` and `:storage :sqlite-memory` are exercised, or document why one mode is intentionally not used in the fixture
- Keep the fixture temporary or generated by a test/helper script; do not commit a large vendored test project unless the implementation benefits clearly outweigh the maintenance cost.
- If docs examples drift from the working fixture, update `docs/library-authoring.md`.

## TASK-007.P3 Done when

- A separate external-style fixture successfully runs tests against this Atom checkout.
- The fixture proves `atom.test.alpha` is consumable from `src/atom/test/alpha.clj` via tools.deps local-root.
- The fixture proves real daemon library activation through `libs.edn`, `sync!`, and `use!`.
- The exact command and result are recorded in the plan Developer Notes or a committed focused test script if one is added.
